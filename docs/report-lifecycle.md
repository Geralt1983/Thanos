# Report Lifecycle

## Purpose
Keep active diagnostics and test reports easy to find without cluttering the repo root.

## Rules
- **Active reports** live at repo root only while they are actionable.
- **Resolved or stale reports** must be moved to `archive/reports/`.
- **Naming**: include a date suffix (`YYYY-MM-DD`) for diagnostics or time-boxed reports.
- **Do not delete** reports; archive them to preserve history.
